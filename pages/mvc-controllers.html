MVC & Controllers Deep Dive

เข้าใจการทำงานประสานกันของ Model, View และ Controller เพื่อสร้างแอปพลิเคชันที่ขยายตัวได้ยั่งยืน

1. ความสัมพันธ์ในแบบ Laravel

  • Model: คุยกับ Database (เช่น User, Post)
  • View: จัดการเรื่องการแสดงผล (Blade)
  • Controller: รับ Request มา แล้วไปบอก Model ให้หาข้อมูล แล้วส่งไปที่ View เพื่อแสดงผล

2. Resource Controllers

แทนที่จะเขียน Method เองทีละอัน เราสามารถใช้คำสั่งสร้างชุด CRUD มาให้ครบเซ็ตในทีเดียว

// สร้าง Resource Controller พร้อม Model
php artisan make:controller PostController --resource --model=Post

คำสั่งนี้จะสร้าง Method หลักๆ มาให้ดังนี้:

  • index - แสดงรายการทั้งหมด
  • create - แสดงฟอร์มสร้างใหม่
  • store - บันทึกข้อมูลลงฐานข้อมูล
  • show - แสดงข้อมูลรายตัว
  • edit - แสดงฟอร์มแก้ไข
  • update - ปรับปรุงข้อมูล
  • destroy - ลบข้อมูล

3. Dependency Injection & Type Hinting

ฟีเจอร์ที่ทำให้ Laravel โดดเด่น คือการที่มันสามารถ "ดึงเครื่องมือ" มาให้เราใช้ได้อัตโนมัติเพียงแค่บอกชื่อ Class

public function store(Request $request, PostService $service)
{
    // Laravel จะส่ง $request และ $service (ถ้าเราสร้างไว้) มาให้เองโดยอัตโนมัติ
    $validated = $request->validate([...]);
    $service->createPost($validated);
    
    return redirect()->back();
}
คำแนะนำ: พยายามทำให้ "Controller ผอม (Skinny Controller)" โดยย้าย Logic ที่ซับซ้อนไปไว้ใน Service Class หรือ Model
บทเรียนก่อนหน้า จัดการฐานข้อมูลต่อ
