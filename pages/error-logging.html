Error Handling & Logging

การจัดการข้อผิดพลาดและบันทึกประวัติการทำงานของระบบเพื่อความปลอดภัยและง่ายต่อการ Debug

1. การบันทึก Logging

เราสามารถใช้ Log facade เพื่อบันทึกข้อมูลและเหตุการณ์สำคัญลงในไฟล์ storage/logs/laravel.log

use Illuminate\Support\Facades\Log;

Log::info('ผู้ใช้ ID 5 เข้ามาแกไขข้อมูล');
Log::warning('พบความพยายามในการเข้าระบบที่ผิดพลาดหลายครั้ง');
Log::error('เกิดข้อผิดพลาดในการดึงข้อมูลจาก API ภายนอก!');

2. การปรับแต่งหน้า Error

เมื่อผู้ใช้เจอหน้า 404 หรือ 500 เราสามารถสร้างดีไซน์ให้เข้ากับแบรนด์ของเราได้เพียงแค่สร้างไฟล์ใน resources/views/errors/

// เช่นการสร้างหน้า resources/views/errors/404.blade.php
<div class="error-page">
    <h1>ขออภัยครับ ไม่พบหน้าที่คุณต้องการ</h1>
    <a href="/">กลับไปหน้าหลัก</a>
</div>

3. การใช้ Try-Catch & Exception

ในการทำงานที่มีความเสี่ยง เช่น การเชื่อมข้อมูลข้าม Server เราควรใช้ Try-Catch เสมอ

try {
    $data = ExternalAPI::fetch();
} catch (\Exception $e) {
    Log::error("API Error: " . $e->getMessage());
    return back()->with('error', 'ระบบขัดข้องชั่วคราว กรุณาลองใหม่ภายหลัง');
}
คำเตือน: อย่าลืมเช็ค APP_DEBUG=false ในไฟล์ .env เวลาใช้งานจริง เพื่อไม่ให้ระบบแสดงความลับของโค้ดแก่คนแปลกหน้าครับ
